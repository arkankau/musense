# Project Description

## App Name
Musense

## The Idea
- An accessibility app that helps Deaf and hard-of-hearing users experience music in a more meaningful way through synchronized visual, tactile, and contextual feedback
- Instead of treating music as only sound, Musense translates songs into dynamic visualizations, vibration-based rhythmic patterns, and simple emotion or energy summaries so users can engage with music in a multisensory way
- Users can either search for songs or use a live listening mode to analyze music in real time.

## Motivation
- Music is one of the most universal forms of expression, yet many existing music apps are designed almost entirely around audio
- We want to make music more inclusive by creating an experience that is accessible beyond hearing alone

## Architecture
- Onboarding / preferences view for calibration and user settings
- Home view where users can start live listening or search for a song
- Live experience View showing synchronized visuals and haptic rhythm feedback
- Song detail view with music metadata, emotion/energy description, and saved preferences
- Saved sessions / library view where users can revisit previously analyzed songs or experiences

## How to Run
- Open the .xcode project
- Connect to a simulator (Mac or iPhone)
